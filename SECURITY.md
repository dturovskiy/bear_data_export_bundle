# Security Policy

## Reporting a Vulnerability

**Please do NOT open a public GitHub Issue for security vulnerabilities.**

If you discover a security issue in this project, please report it **privately** by emailing:

### What to include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### Response SLA

| Action | Timeframe |
|---|---|
| Acknowledgement | **72 hours** |
| Initial assessment | 1 week |
| Fix release (if applicable) | Best effort, depends on severity |

## Scope

This policy applies **only** to the code in this repository (`binance_ohlcv_exporter.py` and associated scripts).

Issues related to the Binance API itself should be reported directly to [Binance](https://www.binance.com/en/support).
